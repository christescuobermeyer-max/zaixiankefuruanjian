# 前端原型一比一复刻设计

## 目标
把 `前端ui原型图/外卖客服桌面助手.dc.html` 复刻为 `apps/desktop` 内可运行、可构建、可测试的 React + Vite 前端。当前阶段聚焦视觉和本地交互还原，作为后续 Tauri/React、Supabase 登录和真实后端 API 对接的基础。

## 范围
- 还原 Windows 桌面背景、左侧桌面图标、底部任务栏、顶部热区和隐藏胶囊。
- 还原 720px 顶部居中客服面板、标题栏、在线状态、Pin/隐藏按钮、圆角、阴影和动画。
- 还原登录态、已登录聊天态、搜索态、流式回复态、空会话态和新建对话弹窗。
- 使用原型内置的示例对话、快捷话术和规则回复，保持本地可交互。
- 使用 React state 管理登录、会话、搜索、发送、流式模拟、弹窗和自动隐藏。

## 暂不实现
- 不接真实 Supabase 登录。
- 不调用后端 API。
- 不实现 Tauri Rust command、capability、真实顶部鼠标热区或系统窗口行为。
- 不实现 Tauri updater 和 OSS 发布。

## 技术方案
- 在 `apps/desktop` 建立 React + TypeScript + Vite 工程。
- UI 分为桌面外壳、顶部面板、登录表单、对话侧栏、聊天主区、新建对话弹窗等组件。
- 样式使用普通 CSS 和 CSS 变量，尽量保留原型中的颜色、尺寸、间距、阴影、动画和状态变量。
- 图标使用内联 SVG，减少依赖并保证与原型视觉一致。
- 测试使用 Vitest + React Testing Library + jsdom，覆盖关键状态和交互。

## 数据与交互
- 初始状态为隐藏面板，显示顶部“客服”胶囊和引导提示。
- 点击胶囊或进入顶部热区展开面板；面板离开后延迟自动隐藏；Pin 状态禁用自动隐藏。
- 登录表单本地校验邮箱和密码，任意非空密码可进入演示聊天态。
- 会话列表包含“川味小厨”“幸福麻辣烫”“老王烧烤”三条示例。
- 搜索框按店铺名过滤会话，展示搜索中、搜索结果和无结果状态。
- 发送消息后添加用户气泡，并根据关键词生成本地客服回复，按字符模拟流式输出。
- 新建对话后插入列表顶部并自动选中，生成欢迎回复。

## 验证
- `pnpm --filter desktop test`
- `pnpm --filter desktop build`
- `python scripts/verify.py`
- 启动 `pnpm --filter desktop dev`，用浏览器检查 1280x800 预览、面板 720x560 布局、登录/聊天/搜索/弹窗/流式状态。
