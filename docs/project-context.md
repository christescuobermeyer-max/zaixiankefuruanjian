# 项目上下文

## 业务需求
- 用户是谁：需要长时间处理外卖订单咨询、催单、退款、缺漏餐和菜品推荐的商家客服人员。
- 要完成的工作：在 Windows 桌面顶部快速唤出一个 AI 客服助手，登录后按店铺/对话维护上下文，并获得流式中文客服回复。
- 核心业务流程：用户登录 Supabase 账号，创建命名对话，输入顾客诉求，后端注入外卖客服 system prompt 和历史上下文，调用云雾 OpenAI 兼容接口流式返回，并把对话保存到 MongoDB。
- 支持的平台：MVP 只支持 Windows 桌面端；后端运行在 Sealos/云服务器。
- MVP 范围：自动隐藏顶部面板、登录鉴权、多对话、纯文本流式聊天、MongoDB 持久化、上下文压缩、Tauri Updater + OSS 更新。
- 明确不做：多租户/团队协作、移动端、语音、图片多模态、本地离线模型。
- 真实操作风险：真实密钥曾在初始资料中出现；自动更新会写 OSS；后端会持有 LLM、MongoDB 和鉴权相关 secret。
- 成功标准：客户端可登录、新建命名对话、流式聊天、保持上下文；后端可鉴权、隔离用户数据、压缩历史、落库；安装包可通过 OSS 子目录更新。
- 最终结果验收标准：按真实 Windows 桌面流程验证面板唤出/隐藏、登录、创建对话、收发消息、历史恢复、上下文压缩和更新检查均达到用户设计文档要求。

## 领域术语
- 店铺名/对话名称：创建对话时输入的名称，用于区分不同客服上下文。
- 顶部热区：屏幕顶部 2px 且位于面板水平范围内的鼠标触发区域。
- Pin 模式：用户固定面板，禁用自动隐藏。
- 上下文压缩：超过 token 阈值后把较早消息摘要为 summary，保留最近轮次进入 LLM 上下文。
