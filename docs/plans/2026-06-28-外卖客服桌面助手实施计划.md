# 外卖在线客服 AI 桌面助手 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 构建一个 Tauri 桌面客户端 + Node/Hono 后端的外卖在线客服 AI 助手，桌面顶部自动隐藏面板、Supabase 登录鉴权、流式 LLM 对话、MongoDB 持久化、token 阈值上下文压缩、阿里云 OSS 自动更新。

**Architecture:** 三层：Tauri（Rust 外壳 + React 前端，Vercel AI SDK + prompt-kit）↔ Node+Hono 后端（JWT 校验、prompt 注入、上下文压缩、Mongo 存储、转发云雾 LLM）↔ MongoDB / 云雾 LLM / 阿里云 OSS。密钥仅在后端，客户端永不接触 LLM Key 与 Mongo。

**Tech Stack:** Tauri 2.x, React+TS+Vite, Vercel AI SDK, prompt-kit, supabase-js, Node+TypeScript+Hono, MongoDB driver, ali-oss, @tauri-apps/plugin-updater, Vitest（后端测试）。

---

## 阶段总览

- **Phase 0**：仓库与工程脚手架、环境变量、`.gitignore`（密钥安全）
- **Phase 1**：后端 —— 健康检查、JWT 中间件、Mongo 连接
- **Phase 2**：后端 —— 对话 CRUD
- **Phase 3**：后端 —— 上下文压缩服务
- **Phase 4**：后端 —— `/api/chat` 流式接入云雾 LLM + 持久化
- **Phase 5**：客户端 —— 登录页（Supabase Auth）
- **Phase 6**：客户端 —— 聊天主面板（AI SDK + prompt-kit）+ 新建对话弹窗
- **Phase 7**：客户端 —— Tauri 自动隐藏面板（热区/状态机）
- **Phase 8**：自动更新（Updater + OSS 发布脚本）
